# simpleERP
ERP是為了讓企業的決策者更完善的監控、管理並協助企業做出正確決策的輔助系統。他整合了企業內部一切流程及資源，在過去企業的資訊系統都是獨立模組，甚至還有手抄紙本方式記錄，這樣的運作模式很難達到資訊共享，資訊不足的狀況容易讓決策者做出錯的決定，而ERP是將企業中的所有資訊全部整合起來，其目的是完整的紀錄所有資訊進而為決策者提供即時且有效的資訊，達到資源的更有效利用。

&emsp;
## simpleERP提供的服務?

simpleERP是以ERP及買賣業作為藍本製作的服務網站，透過使用者群組及職等實現公司簽核流程，主要的功能如下:

* 表單新增
    * 側邊的目錄，新增各項表單，進行公司各項營業活動的紀錄。
    ex:廠商基本資料、訂單資料新增、優惠單價新增
    * 依照使用者在的職等群組，控制使用者所能新增的表單類型


* 表單審核
    * 簽核的站別主要拆分成四個階段start、middle、final、complete
    * 建立的相關申請單，在建立時會依照填單人職等，及其對應的簽核站別儲存於資料庫中
    * 使用者在登入時會依照其職等及部門，搜索資料庫有無在該站別的申請單
    * 提供駁回及核准兩個按鈕讓申請單更新站別及申請單狀態
    * 使用者可於會員頁面上傳印鑑，審核過後的表單會顯示其對應的印鑑圖片
    * 特定表單依照條件設定，可跳過老闆簽核站別，節省最高權限主管的時間
        舉例(1)廠商資料申請單，付款條件為立即付款，則簽至經理站別即審核完成。
        舉例(2)產品資料申請一律都簽至經理即算審核完成。


* 歷史紀錄查詢
    * 填寫完新增表單後，各使用者皆可於申請單查詢選單中，可查詢自己或是其他使用者新增的表單。
    * 利用各申請單查詢頁面及搜尋條件來查詢各表單
    ex:使用日期、廠商名稱、申請單號、申請單狀態及負責業務等來查詢要找的申請單
    * 若申請單尚未跑完簽核流程，僅會在該申請單對應站別的簽核者顯示駁回、核准及新增簽核意見的按鈕


* 圖表分析
    * 設定查詢的日期及產出的可視化圖片類型，讓使用者可以利用產生的摺線圖或圓餅圖進行分析ex:產品趨勢圖

# 使用技術

### 前端
* HTML
* CSS
* Javascript
* chart.js


### 後端
* Python Flask 架設 application server
* Flask_JWT_Extended 套件管理使用者登入狀態
* Docker 將伺服器部屬至AWS EC2上
* Nginx 架設 Web server 進行網域的反向代理及安裝 SSL 證書安裝將域名等訊息用您的伺服器金鑰做加密
* 產品及印鑑照片儲存於AWS S3，並搭配cloudfront做CDN加速圖片的讀取速度
* MySQL儲存個表單的資訊，並使用index及foreign key加快查詢速度，使
